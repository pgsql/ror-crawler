Department Description

Solaris File Protocols and Interoperability is responsible for Solaris file sharing protocols including NFS and SMB/CIFS and the Lustre high-performance, scalable clustered storage file system.  This development and test is leading innovation in shared, network file systems and interoperability between diverse platforms, protocols, and interconnects.  We contribute technology to Solaris as well as into Oracle's Open Storage systems including the ZFS Storage Appliance. Our focus and passion is on delivering robust, enterprise-ready customer solutions and providing technology leadership in standards-based file protocol.

Provide technical leadership to other software developers. Specify, design and implement modest changes to existing software architecture to meet changing needs.

Additional Details

This is a position for a mid-level to senior software engineer who will participate on a team developing Solaris CIFS/SMB file sharing services. This individual will be focusing on the CIFS/SMB client, smbfs, the Solaris component that is used to access remote file shares on Windows Servers. This module is also a core part of the Windows interoperability framework in that it is responsible for Active Directory interactions such as domain join.

This position prefers 4 or more years of software development experience in the C/Solaris/Unix/Linux environments. It is desirable the candidate has design/development experience in file systems or file sharing protocols such as CIFS/SMB or NFS.

The candidate is expected to demonstrate solid software engineering practices: complete and thoroughly considered designs, mature coding patterns, and methodical debugging techniques. To succeed in this role, you'll need a strong systems programming background, a knack for debugging and troubleshooting, and the ability to thoroughly learn new technology areas
